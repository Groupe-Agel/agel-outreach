2 Must-haves (legal & safety) Backlog

Build these before scaling outreach volume.

S–M Unsubscribe link & opt-out page

Legal requirement under GDPR (EU), CAN-SPAM (US), CASL (Canada). Add an {{unsubscribe_url}} auto-variable, a public /u/[token] page (no auth) that records the opt-out, a global suppression list, and a refusal at send time.

S Suppression list (do-not-contact)

Global blocklist regardless of which list a contact is on. Use cases: bounces, complaints, unsubscribes, manual "do not contact" flags.

S Daily send-rate limit per user

Hostinger shared mailboxes cap at ~300/day. Exceeding can suspend the whole account. Configurable per-user cap, midnight reset in the user's timezone, friendly refusal with remaining-quota message.

M GDPR data-subject request handler

When a contact asks "what data do you have on me?", generate a JSON export of every row + every campaign that sent to that address.

S Privacy policy & footer link

"Privacy" link in the email footer pointing to your policy hosted on agelpartners.com.

3 Productivity wins Backlog

Pay back daily for the team.

S Drafts

The DRAFT status already exists. Expose it: save a half-finished campaign and resume later.

S Duplicate a campaign

"Duplicate" button that copies template/subject/sender — but starts with empty contacts.

M List editing

Rename, edit description, add/remove individual members on an existing list.

M Shared lists across team

Mark a list as "shared" so colleagues can use it in their own campaigns.

S Bulk actions

Multi-select campaigns to archive, delete, or duplicate at once.

M Search & filter

Free-text search by name; filter by status, date range, sender.

S Keyboard shortcuts

⌘K command palette, ⌘N new campaign, J/K to navigate lists.

M Onboarding tour

First-login walkthrough: SMTP setup → first template → first contacts → first test send.

4 Engagement & analytics Backlog

Turn outreach from a black box into a measurable funnel.

M Per-campaign analytics dashboard

Open rate, delivery rate, fail rate, reply rate. Timeline of opens over the first 72 hours. Per-recipient breakdown.

M Click tracking

Wrap outbound links in tracking redirects, log clicks per recipient.

S Open tracking pixel

1×1 transparent GIF served per recipient. Many corporate clients block it — useful but not authoritative.

L Best send-time recommendations

After enough data, suggest send times based on when this audience usually opens.

L A/B testing

Send variant A to 50%, variant B to 50%. Auto-pick the winner by open rate after N hours, send to the rest.

M Campaign comparison

Side-by-side open/click/reply metrics across two campaigns.

S CSV export

Per-campaign export of recipient statuses + merge data + timestamps.

M Live dashboard

Aggregate stats this week / month: campaigns sent, recipients reached, average open rate, top templates.

5 Reply handling Backlog

The single biggest UX win for an outbound team.

L IMAP reply tracking

Poll each user's IMAP inbox; match incoming messages to outbound campaigns by Message-ID / In-Reply-To. Update recipient status to REPLIED.

L In-app inbox view

Per-campaign timeline of replies — read, archive, mark as won/lost.

L Auto-categorize replies

Classify as interested / not-interested / out-of-office / auto-reply / human.

S Auto-pause on reply

If a contact replies to step 1 of a sequence, don't send step 2.

S Reply templates

Small library of quick-reply snippets per user.

6 Deliverability & quality Backlog

Keep emails out of spam folders.

M Pre-send spam-score check

Analyze subject + body for spam triggers (ALL CAPS, excessive !, "free", "click here"). Show a 0–10 score before send.

S Template linting

Warn if MJML references variables not in contacts, has broken closures, or missing alt-text on images.

M Domain authentication wizard

Walk users through SPF, DKIM, DMARC setup. Verify by DNS lookup.

L Deliverability score per user

Track bounce rate, complaint rate, open rate. Show 0–100 score; warn if degrading.

XL Inbox preview (Litmus-style)

Render the email in Gmail web, Apple Mail, Outlook desktop, mobile clients. Requires third-party API ($).

S Image alt-text validation

Refuse to send if <mj-image> is missing alt. Hurts accessibility and deliverability.

7 Sequences & automation Backlog

L Multi-step sequences

L Conditional logic

"If opened step 1 but didn't click, send X; if didn't open, send Y."

M Drip campaigns

"Send at 9 AM local time" per recipient, computed from a timezone column.

9 Contact management Backlog

L Unified contact profile

Click an email address anywhere → see every list, every campaign, every reply, all merge data. A mini CRM.

M Custom fields

Beyond name/organization, let users define their own fields per list.

S Tags

Tag contacts (vip, partner, cold-lead) and filter lists by tag.

M Contact deduplication

Cross-list dedup with a "which lists" indicator.

L Import from Google / Outlook

OAuth into Google Contacts or Microsoft Graph and pull contacts.

10 Templates & content Backlog

M Version history

Each save creates a version; rollback to any prior version.

S Variables manager

Panel showing every variable used by the current template with sample values for preview.

M Image hosting / asset library

Upload images to object storage (Clever Cellar / S3), paste URLs into MJML. Optional toolbar button for one-click insert.

M Multi-language variants

A template can have FR, EN, AR variants. Send the right one based on a contact's locale.

S Template categories

Group templates: "Cold outreach", "Welcome series", "Newsletters".

M AI-assisted writing

"Rewrite this for a warmer tone" / "Make it shorter" via Claude API. Saves drafts as suggestions, never sends without confirmation.

12 Future ideas Idea

  • Per-recipient AI personalization — generate 3 unique opening lines per contact from LinkedIn / company data.
  • Inbox warm-up — slowly ramp send volume on a new mailbox to build sender reputation.
  • Native mobile app — view replies and stats from a phone.
  • Voice notes in emails — record a short audio clip, hosted, included as a link with preview thumbnail.
  • Video personalization — auto-generate a Loom thumbnail with recipient's name overlaid.
  • Custom tracking domains — links.agelpartners.com/c/xxx instead of generic URLs.
  • Public campaign report links — share a read-only stats URL with non-account stakeholders.
  • Real-time collaboration — multiple people editing the same template at once.
  • AI subject-line suggester — generate 5 variants, predict open rate for each.

13 How to read this catalogue

For planning sprints: pick from §2 (legal) first, then §3 (productivity) to compound returns daily. §5 (reply handling) is the highest-leverage feature for outbound sales — schedule it when you have a 1–3 day window.

For prioritization arguments: effort tags + impact category give a quick way to argue tradeoffs. "An A/B test (L) vs a daily-send-limit (S)" — the S wins almost every time.

For onboarding: skim §1 to understand what's already built before adding to it.

Open a ClickUp ticket in the Outreach list when you start work on any feature here. Cross-reference the section + feature name in the ticket description.
